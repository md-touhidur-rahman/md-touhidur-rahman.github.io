Traffic Light State Detection with CNN (CARLA Dataset)

CNN-based traffic light classifier trained on simulated CARLA images to distinguish Red, Green, Yellow, and Back states — a step toward perception modules for autonomous and connected vehicles.

Objective

To simulate real-time perception in an autonomous driving pipeline by classifying traffic light states from RGB images. The model outputs one of four states: red, green, yellow, or back/off. This kind of model can later be plugged into a ROS or CARLA-based control stack.

Dataset

  • Source: CARLA simulator traffic light frames (manually labeled)
  • Classes: red, green, yellow, back
  • Structure: pre-split into train/ and val/ folders
  • Format: RGB images, resized to 64×64

Model Architecture

Implemented in TensorFlow/Keras:

  • Input: 64x64x3 RGB image
  • Stack of Conv2D → ReLU → MaxPooling blocks
  • Dropout before dense layers to reduce overfitting
  • Final dense layer with Softmax for 4-class classification

Results

  • Validation Accuracy: 99.22%
  • F1-scores: near-perfect across all four classes
  • Includes confusion matrix and sample predictions (see sample_outputs/)

Evaluation

The model was evaluated using standard classification metrics:

  • Classification report (precision, recall, F1)
  • Confusion matrix to check per-class performance
  • Visual inspection of predictions on unseen validation images
